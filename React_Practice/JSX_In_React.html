                                        üìù Essential Rules of JSX in React


=======================================================================================================================
    ->JSX (JavaScript XML) has a strict set of rules that distinguish it from standard 
      HTML. Adhering to these rules is crucial for writing valid React code.

     1. Single Root Element:

            ->Every JSX block must return a single root element. You cannot 
              have two top-level elements adjacent to each other.

            ‚ùå Incorrect:
                    return (
                        
                        <h1>Title</h1>
                        
                        <p>Content</p>
                        );

            ‚úÖ Correct (Using a <div>):
                    return (
                    

                        
                        <h1>Title</h1>
                        
                        <p>Content</p>
                    

                    );

            ->Use the shorthand Fragment syntax (<>...</>) to group elements 
              without adding an extra node to the DOM.

                    return (
                        <>
                            
                        <h1>Title</h1>
                        
                        <p>Content</p>
                        
                         </>
                        );


    2. Case and Naming Conventions

        ‚¨§ HTML Tags vs. Components
            HTML Tags (lowercase): 
            Standard HTML tags must be written in lowercase (e.g. <div>,<span>,<img>).

            Custom Components (Capitalized): 
            React custom components must start with a capital letter (e.g.,<MyComponent>,<UserProfile>). React treats 
            lowercase tags as standard HTML/SVG elements.

        ‚¨§ Attribute Naming
            ->Attributes are written in camelCase, following JavaScript conventions, not kebab-case (like in HTML).

                üëâclass becomes className: 
                    ->This is the most common difference, as class is a reserved keyword in JavaScript.
                üëâfor becomes htmlFor: 
                    ->Used for associating labels with form elements.
                üëâonclick becomes onClick.
                üëâtabindex becomes tabIndex.

    

    3. Closing Tags
            ->All tags must be explicitly closed. This applies to both parent tags and self-closing tags.
            
            ‚ùå Incorrect (Self-closing tags):
                    <img src="path/to/image.jpg">
                    


            ‚úÖ Correct (Self-closing tags): Self-closing tags must end with a forward slash (/).
                    <img src="path/to/image.jpg" alt="Description" />
                    


            ‚úÖ Correct (Paired tags): Standard tags must have an opening and a closing pair.
                    <button>Click Me</button>


    4. Embedding JavaScript Expressions
            ->You can embed any valid JavaScript expression inside JSX by wrapping it in curly braces ({}).

            ‚¨§ Expressions Only: 
                    ->You can use variables, function calls, arithmetic operations, and object properties.
            ‚¨§ No Statements: 
                    ->You cannot use JavaScript statements like if...else blocks or for loops directly inside JSX. You must use 
                      ternary operators, logical operators, or map functions instead.


                    const user = { name: 'Alice', age: 30 };
                    const isLoggedIn = true;

                    return (
                    <div>
                        {/* Embedding a variable */}
                        <h1>Hello, {user.name}!</h1>

                        {/* Embedding an arithmetic operation */}
                        <p>The total is: {5 + 3}</p>

                        {/* Embedding a conditional expression (Ternary Operator) */}
                        <p>Status: {isLoggedIn ? 'Logged In' : 'Logged Out'}</p>
                    </div>
                    );

    5. Comments

        ->JSX treats everything outside of curly braces as static text. To add a 
        comment inside the JSX element structure, you must wrap the standard 
        JavaScript comment syntax (/* ... */) within curly braces.

            ‚¨§ Inside child elements:
                    <div>
                    {/* This is a single-line comment */}
                    <p>Content</p>
                    {/*
                        This is a multiline
                        comment block.
                    */}
                    </div>  
                    
            ‚¨§ Inside attributes (not common):
                    <input
                    type="text"
                    // The comment must be on a new line when inside attributes
                    placeholder="Enter your name"
                    />

    6. Attribute Values

        ->String Literals: Use double quotes for static string values.
                <button id="my-button" type="submit">Send</button>
        ->JavaScript Expressions: Use curly braces ({}) to pass any non-string values (numbers, booleans, 
          objects,functions, variables). Do not use quotes around the curly braces.
                const count = 10;
                const handleClick = () => alert('Clicked');

                <MyComponent
                count={count}           /* Number variable */
                isActive={true}         /* Boolean literal */
                onClick={handleClick}   /* Function reference */
                style={{ color: 'red', fontSize: '16px' }} /* Object for inline styles */
                />

                                                Basic Concepts

===========================================================================================================

    ‚úÖ Embedding JavaScript Expressions in JSX:-
            ->If you want to print or calculate something inside JSX, wrap it in { }.

                function App() {
                const name = "Jay";
                const num1 = 10;
                const num2 = 20;

                return (
                    <div>
                    <h2>Hello {name}</h2>
                    <p>Sum = {num1 + num2}</p>
                    </div>
                );
                }

    ‚úÖ Using Attributes in JSX:-
            ->JSX attributes look like HTML but follow camelCase.

                | HTML Attribute | JSX Alternative |
                | -------------- | --------------- |
                | `class`        | `className`     |
                | `onclick`      | `onClick`       |
                | `tabindex`     | `tabIndex`      |

            ->example:

                function App() {
                return (
                    <button className="btn" onClick={() => alert("Clicked!")}>
                    Click Me
                    </button>
                );
                }

    ‚úÖ Conditional Rendering in JSX:-
            ->Example 1: Ternary operator
                    function App() {
                    const isLoggedIn = true;

                    return (
                        <h2>{isLoggedIn ? "Welcome User" : "Please Login"}</h2>
                    );
                    }
            ->Example 2: Logical AND (&&)
                    function App() {
                    const showMessage = true;

                    return (
                        <div>
                        {showMessage && <p>Hello JSX!</p>}
                        </div>
                    );
                    }

    ‚úÖ Inline Styling in JSX
            üîπ Style in JSX uses an object, not a string.
            üîπ Keys must be camelCase.

                    function App() {
                    const myStyle = {
                        color: "white",
                        backgroundColor: "purple",
                        padding: "10px",
                        borderRadius: "10px"
                    };

                    return <h2 style={myStyle}>Styled with JSX</h2>;
                    }

    ‚úÖ List Rendering in JSX (Looping with map())
            function App() {
            const fruits = ["Apple", "Mango", "Banana"];

            return (
                <ul>
                {fruits.map((item, index) => (
                    <li key={index}>{item}</li>
                ))}
                </ul>
            );
            }

    ‚úÖ JSX with Components
            ->Parent to Child Props Example
                function Card(props) {
                return <h2>Hello {props.name}</h2>;
                }

                function App() {
                return (
                    <div>
                    <Card name="Jay" />
                    <Card name="React" />
                    </div>
                );
                }
