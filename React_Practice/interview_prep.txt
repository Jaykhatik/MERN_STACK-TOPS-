50 Professional CSS Interview Questions & Answers
I kept answers concise but professional ‚Äî suitable for interview settings.

1. Q: What is the CSS box model?
A: The box model describes an element‚Äôs layout as content ‚Üí padding ‚Üí border ‚Üí margin. Use box-sizing: border-box to include padding and border in the width/height for predictable sizing.

2. Q: Explain box-sizing and why border-box is recommended.
A: box-sizing: border-box makes width and height include padding and border, avoiding surprise increases in element size and simplifying responsive calculations.

3. Q: Difference between em and rem.
A: em is relative to the current element‚Äôs font-size; rem is relative to the root (html) font-size. rem is typically preferred for predictable scaling.

4. Q: When would you use flex vs grid?
A: Use Flexbox for one-dimensional layouts (row/column) and Grid for two-dimensional layouts (rows + columns). Many layouts combine both.

5. Q: What is the cascade in CSS?
A: The cascade resolves conflicts using importance (!important), specificity (inline > id > class > element), and source order (later declarations override earlier ones).

6. Q: Explain CSS specificity.
A: Specificity is a weight calculation: inline styles (1,0,0,0), IDs (0,1,0,0), classes/attributes/pseudo-classes (0,0,1,0), elements/pseudo-elements (0,0,0,1). Higher specificity wins.

7. Q: How does position: absolute determine its reference point?
A: It is positioned relative to the nearest ancestor with a non-static position (relative, absolute, fixed, sticky). If none, it uses the initial containing block (viewport for fixed).

8. Q: What is a stacking context?
A: A stacking context is an isolated layer that determines z-index ordering. Created by root, positioned elements with z-index, opacity < 1, transform, and other properties. Elements in different stacking contexts don‚Äôt compare z-index directly.

9. Q: How do CSS variables differ from preprocessor variables?
A: CSS custom properties are native, dynamic, and inherited at runtime (modifiable via JS). Preprocessor variables (Sass, Less) are compiled at build time and cannot be changed at runtime.

10. Q: How does flex: 1 1 auto work?
A: Shorthand for flex-grow, flex-shrink, and flex-basis. flex:1 commonly means item will grow to fill available space and shrink as needed.

11. Q: Describe grid-template-areas.
A: A semantic way to define grid layout by naming areas in a matrix. Items are assigned to areas for readable layouts.

12. Q: What is min-content, max-content, and fit-content?
A: Intrinsic sizing keywords: min-content is minimal width without overflow, max-content is full-width unwrapped content, fit-content() clamps to available space. Used for advanced sizing.

13. Q: How do clamp(), min(), and max() help responsive design?
A: clamp() sets a value bounded between min and max with an ideal value (e.g., clamp(1rem, 2.5vw, 2rem)), enabling fluid scaling while preventing extremes.

14. Q: What is object-fit and when to use it?
A: Controls how replaced content (images/videos) fills its box: cover, contain, fill, none, scale-down. Use to maintain aspect ratios without distortion.

15. Q: Explain CSS transitions vs animations.
A: Transitions animate property changes between states and need a trigger (e.g., hover). Animations (@keyframes) are standalone sequences that can run automatically with finer control.

16. Q: What is will-change and when should you use it?
A: Hints the browser which properties will change to enable optimizations (e.g., promote to their own compositor layer). Use sparingly because overuse can increase memory usage.

17. Q: What are CSS logical properties?
A: Properties like margin-block-start, padding-inline-end that adapt to writing modes and directionality (LTR/RTL), improving internationalization.

18. Q: When should you avoid position: fixed?
A: Avoid when content size or viewport changes make fixed elements obstructive (mobile keyboards), or when many fixed elements cause performance issues.

19. Q: How to make accessible focus styles?
A: Provide clear :focus-visible outlines, maintain contrast, and ensure keyboard users can navigate and see focus. Avoid removing outlines without replacement.

20. Q: What‚Äôs the problem with display: none for accessibility?
A: display: none hides content from all users and assistive technologies. If hiding visually only, consider using ARIA methods and maintain accessibility semantics.

21. Q: Explain prefers-reduced-motion.
A: A media query to respect user motion preferences. When set to reduce, disable or simplify non-essential animations.

22. Q: How do you debug a layout problem?
A: Use DevTools: inspect computed styles, toggle CSS properties, add temporary outlines, and check box model in layout panel.

23. Q: How to avoid layout thrashing?
A: Minimize read/write cycles to DOM, batch DOM reads and writes, use transforms for animations rather than layout properties (left/top), and use requestAnimationFrame for JS-driven animations.

24. Q: Why use box-shadow carefully?
A: Expensive to paint if used on many elements; can cause repaints and performance degradation on scrolling.

25. Q: How does mix-blend-mode affect performance?
A: May trigger additional compositing and repaints, so use only where visually important. Use isolation: isolate when necessary to limit blending.

26. Q: What‚Äôs the difference between visibility: hidden and display: none?
A: visibility: hidden hides element but keeps its layout space; display: none removes it from layout flow entirely.

27. Q: How would you implement a responsive image approach?
A: Use srcset and <picture> with different image sizes and sizes attribute, or use image-set()/responsive images strategy to serve optimized images per device.

28. Q: What are CSS media queries and how are they used?
A: Conditional blocks applying styles based on environment (viewport width, resolution, reduced motion, print). Fundamental for responsive design.

29. Q: What are pseudo-elements used for?
A: Insert generated content (::before, ::after), style parts of elements (::first-line), and create shapes or decorative elements without extra markup.

30. Q: Explain contain property.
A: contain limits browser work for an element (layout, paint, size, style containment), improving performance.

31. Q: When to use translate vs changing left/top?
A: Use transform: translate() for performant animation (GPU-accelerated, avoids layout), whereas left/top triggers layout and paint.

32. Q: How to manage critical CSS?
A: Inline above-the-fold (critical) CSS in HTML head and lazy-load the rest to reduce first paint time. Use tools to extract critical CSS.

33. Q: What is subpixel rendering and how does CSS affect it?
A: Rendering elements at fractional pixels ‚Äî fonts and transform translations can result in blurred pixels; use transform-translate with integers or adapt line-height/font-smoothing for crispness.

34. Q: What is the CSSOM?
A: CSS Object Model ‚Äî the API representation of CSS accessible to JavaScript for runtime manipulations.

35. Q: How do vendor prefixes work and when to use them?
A: Early implementations required prefixes (-webkit-, -moz-, -ms-). Use Autoprefixer to add necessary prefixes. Avoid new code depending on prefixes unless necessary.

36. Q: What are CSS shorthands? Provide examples.
A: Combine multiple properties: margin: 10px 20px; border: 1px solid #000; transition: all .3s ease; Efficient and concise if used carefully.

37. Q: How do you approach theming with CSS?
